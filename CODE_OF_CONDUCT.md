# Code Of Conduct

## Our Standard

We want this project to be a respectful, technically serious place for collaboration and discussion.

Examples of behavior that contribute to a positive environment include:

- being respectful in disagreement
- assuming good intent while discussing technical trade-offs
- giving actionable, reproducible feedback
- focusing criticism on code, behavior, and evidence
- being welcoming to contributors with different backgrounds and experience levels

Examples of unacceptable behavior include:

- harassment, abuse, or personal attacks
- discriminatory or exclusionary language
- doxxing or publishing private information without permission
- trolling, baiting, or intentionally disruptive behavior
- repeated bad-faith issue or pull request activity

## Scope

This code of conduct applies in:

- GitHub issues
- pull requests
- project discussions
- maintainer or contributor communications that are directly tied to this repository

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, and other contributions that do not align with this code of conduct.
