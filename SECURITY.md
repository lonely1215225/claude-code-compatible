# Security Policy

## Supported Versions

This repository is moving quickly and does not currently maintain long-lived release branches.

| Version | Supported |
| --- | --- |
| Latest `main` branch | Yes |
| Older snapshots / forks | No |

## Reporting a Vulnerability

Please report security issues privately by email:

- `lonely121522521@gmail.com`

When reporting a vulnerability, include:

- a short description of the issue
- the affected platform and Bun version
- the active provider or backend, if relevant
- reproduction steps or a minimal proof of concept
- any suggested mitigation if you already have one

Please do not open a public GitHub issue for unpatched security problems.

## Response Expectations

- Initial acknowledgement: best effort, usually within 72 hours
- Status updates: best effort while triage or mitigation is in progress
- Fix timing: no guaranteed SLA

Because this is a reverse-engineered compatibility project, some issues may depend on upstream behavior, disabled feature-flag paths, or provider-specific integrations. Reports that include exact environment details are much easier to validate.
